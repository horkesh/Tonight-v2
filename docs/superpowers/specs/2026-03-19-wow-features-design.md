# Tonight — Wow Features Design Spec

Six features to elevate the date experience. Confirmed decisions from brainstorming.

---

## 1. Location Image Evolution

**Purpose:** The venue feels alive — the scene shifts as the mood deepens.

**Trigger:** Every 3 rounds (round 3, 6, 9...) when the host returns to HubView.

**Flow:**
1. In `useNarrativeFlow`, when `round % 3 === 0` and `lastLocationImageRound !== round`, fire `generateLocationTransition()`.
2. New Gemini function returns `{ narrative: string, imagePrompt: string }`. Inputs: vibe stats, round, conversation highlights (last 3 entries), current location environment prompt.
3. LocationWindow shows the narrative line as a fade-in overlay ("The bartender dims the lights...") for ~3 seconds.
4. Behind it, `generateLocationImage()` fires with the evolved prompt. New image crossfades in via existing AnimatePresence.
5. Updated `generatedImage` + `environmentPrompt` saved to dateContext, synced via P2P.

**State:** Add `lastLocationImageRound: number` to `gameState`.

**Files:** `hooks/useNarrativeFlow.ts`, `services/geminiService.ts`, `components/LocationWindow.tsx`, `store/gameState.ts`.

## 2. Whisper Mode

**Purpose:** A physical lean-in moment that bridges digital and real-world interaction.

**Trigger:** Phone tilt > 55° (already detected as "pour" in `useDeviceSensors`). Repurpose or add a separate callback with distinct debounce.

**Behavior:**
1. Screen dims (overlay with `bg-black/40`), text shrinks subtly, heartbeat pulse animation on background.
2. After a 1.5s beat, a whisper dare appears — a short, bold statement generated by AI or drawn from a curated pool. Examples: "Tell her the first thing you noticed about her." "Ask her what she's afraid to ask you."
3. No buttons, no answer tracking. Just text on screen. The dare is meant to be said out loud.
4. Tapping anywhere or tilting phone back dismisses. Normal view resumes.
5. Cooldown: 60 seconds between whisper triggers to prevent spam.

**AI vs curated:** Use a curated pool of ~20 dares stored in `constants.ts`. Faster, no API call, always available. Can be enhanced later with AI-generated context-aware dares.

**Scope:** Host-only (only the host's phone triggers this). Not synced to partner — it's private intel.

**Files:** `hooks/useDeviceSensors.ts` (new callback), `components/WhisperOverlay.tsx` (new), `constants.ts` (dare pool), `App.tsx` (mount overlay).

## 3. Shared Playlist Activity

**Purpose:** Build a playlist together. The overlap reveal creates a genuine connection moment.

**Flow:**
1. Added as a new activity in `ACTIVITIES` constant: `{ id: 'playlist', title: 'Our Playlist', description: 'Build tonight\'s soundtrack together.', icon: '🎵' }`.
2. Host triggers via activity menu or narrative suggestion.
3. AI generates 8 songs based on: both personas, vibe stats, conversation log, location mood. Each song has `{ title: string, artist: string, vibe: string }`.
4. Both players see the same 8 songs. Each picks 3 (tap to select, tap again to deselect). Submit when 3 are chosen.
5. Choices sync via existing `SYNC_ACTIVITY_CHOICE` — but extended: instead of a single number, send an array of 3 indices.
6. Reveal: songs animate in one by one. Matched songs (both picked) get a special highlight with "You both felt this one" label. Unmatched songs show who picked them.
7. Match count determines result: 0 = "Different wavelengths", 1 = "Found common ground", 2 = "Tuned in", 3 = "Same frequency".

**Data types:**
```typescript
interface PlaylistData {
  songs: { title: string; artist: string; vibe: string }[];
  subjectId?: undefined; // Not subject-based — both participate equally
}
```

**Sync approach:** Playlist uses a NEW message type `SYNC_PLAYLIST_CHOICE` with `payload: { userId: string; choices: number[] }` — separate from the existing single-number `SYNC_ACTIVITY_CHOICE`. This avoids breaking existing activities. Register a new callback in `registerActivityCallbacks` for playlist choices.

**Type updates:**
- Add `'playlist'` to `ActivityId` union in `types.ts`.
- Add `| { type: 'playlist'; data: PlaylistData }` to `ActivityPayload` union.
- Add `| { type: 'SYNC_PLAYLIST_CHOICE'; payload: { userId: string; choices: number[] } }` to `NetworkMessage` union.
- Store playlist choices separately: `playlistChoices: Record<string, number[]>` in `aiState` (not in `activityChoices`).

**Files:** `constants.ts` (activity entry), `types.ts` (PlaylistData, ActivityPayload, ActivityId, NetworkMessage), `services/geminiService.ts` (generatePlaylistSongs), `hooks/useAiActions.ts` (playlist handler + broadcast), `hooks/useNetworkSync.ts` (SYNC_PLAYLIST_CHOICE handler), `hooks/useBroadcastingState.ts` (broadcastPlaylistChoice), `components/views/PlaylistView.tsx` (new), `store/aiState.ts` (playlistData + playlistChoices), `App.tsx` (lazy load + route).

## 4. "What I Noticed" Card

**Purpose:** Mid-date private intel. One devastating AI observation about her, shown only to the host.

**Trigger:** After round 4, when host returns to HubView. One-shot — only shown once per session.

**Flow:**
1. In `useNarrativeFlow` or a new `useInsightCard` hook, check: `round >= 4 && !insightShown`.
2. Call `generatePartnerInsight()` — new Gemini function. Inputs: partner persona (traits, secrets, memories), conversation log, vibe stats, prompt context.
3. Returns a single string — one sentence, max 25 words. Observational, not flattering. Examples: "She deflects with humor when the question gets close to what she actually wants." "She's testing whether you'll match her energy or fold."
4. Displayed as a brief overlay/toast — frosted glass card that slides in from bottom, auto-dismisses after 8 seconds or on tap. Subtle rose accent.
5. Not synced to partner. Not stored in conversation log. Ephemeral.

**State:** Add `insightShown: boolean` to a local ref (not Zustand — doesn't need persistence or sync).

**Files:** `hooks/useNarrativeFlow.ts` (trigger logic), `services/geminiService.ts` (generatePartnerInsight), `components/InsightCard.tsx` (new), `App.tsx` (mount).

## 5. End-of-Night Letter

**Purpose:** A cinematic closing artifact. An AI-written letter summarizing the evening, styled as handwritten.

**Trigger:** Generated alongside the intelligence report in `finalizeReport()`. Shown after the IntelligenceBriefing modal closes.

**Flow:**
1. In `useAiActions.finalizeReport()`, fire all three in parallel: `Promise.all([generateIntelligenceReport(), generateEndOfNightLetter(), generateFollowUpText()])`. All share the same inputs — no sequential dependency except `generateFollowUpText` receives the letter for tone consistency (so letter must resolve first, then text; report is independent). Use: `report` in parallel with `letter.then(text)`.
2. New Gemini function. Inputs: vibe stats, partner persona, conversation log highlights, rating, location, prompt context.
3. Returns `{ salutation: string, body: string, signoff: string }`. Tone: warm, specific, references real moments from the conversation. ~100 words max. Written as if from a close friend who watched the whole date.
4. Stored in `aiState` alongside the intelligence report.
5. After IntelligenceBriefing closes, a new `LetterView` slides in — full-screen, dark background, the letter rendered in serif italic with generous spacing. Subtle paper texture or slight rotation for handwritten feel.
6. Bottom: "Copy" button (copies plain text) and "Continue" to proceed.
7. Also saved to date history alongside the report.

**Files:** `services/geminiService.ts` (generateEndOfNightLetter), `hooks/useAiActions.ts` (call in finalizeReport), `store/aiState.ts` (letter state), `components/LetterView.tsx` (new), `App.tsx` (flow after briefing closes), `types.ts` (LetterData type).

## 6. Post-Date Text Suggestion

**Purpose:** One perfect follow-up text. No options, no explanation. Just confidence.

**Trigger:** Shown after the end-of-night letter is dismissed.

**Flow:**
1. Generated alongside the letter in `finalizeReport()` — call `generateFollowUpText()`.
2. New Gemini function. Inputs: same as letter + the letter itself (for tone consistency). Returns a single string, 1-2 sentences, SMS-length.
3. The text references a specific moment from the conversation — not generic.
4. Shown on a minimal screen after the letter: the text in a chat-bubble style, a "Copy" button, and "End Session".
5. Tapping "Copy" copies to clipboard. "End Session" triggers `clearSession()`.
6. Also saved to date history.

**Files:** `services/geminiService.ts` (generateFollowUpText), `hooks/useAiActions.ts` (call in finalizeReport), `store/aiState.ts` (followUpText state), `components/TextSuggestionView.tsx` (new), `App.tsx` (flow after letter).

```
Hub → Rate → [generating...] → Intelligence Briefing → Letter → Text Suggestion → End Session
                                                                                         ↓
                                                                                    clearSession()
```

Report generated in parallel with letter→text chain during the "Compiling Final Dossier..." loading state. No additional wait between screens.

## Error Handling

All new Gemini functions follow the existing retry-with-backoff pattern in `geminiService.ts`.

| Feature | On AI failure |
|---------|---------------|
| Location evolution | Skip silently — keep current image, don't show narrative overlay |
| Whisper mode | N/A — curated pool, no AI call |
| Playlist songs | Flash "Activity failed — returning to hub", go to hub (matches existing pattern) |
| Insight card | Skip silently — don't show the card |
| End-of-night letter | Use fallback: `{ salutation: "Tonight", body: "The words escaped before the ink could catch them.", signoff: "—" }` |
| Follow-up text | Use fallback: `"I had a great time tonight."` — generic but functional |

Letter and text failures should NOT block the end-session flow. If either fails, skip that screen and proceed to the next.
